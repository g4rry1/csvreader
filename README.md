# csvreader

Программа читает CSV-таблицу, вычисляет значения ячеек с формулами и выводит результат в том же CSV-формате.

## Формат входного файла

Первая строка — заголовок: пустая ячейка, затем имена столбцов.  
Остальные строки — данные: номер строки, затем значения ячеек.

Ячейка может содержать:
- целое число (например, `42`, `-7`)
- формулу вида `=ARG1OPARG2`, где аргумент — число или адрес ячейки (`ColNameRowNum`, например `A1`, `Cell30`), а оператор — `+`, `-`, `*`, `/`
- пустое значение

```
,A,B,Cell
1,1,0,1
2,2,=A1+Cell30,0
30,0,=B1+A1,5
```

Поддерживаются окончания строк LF, CRLF и CR.

## Сборка

Требуется CMake ≥ 3.10 и компилятор с поддержкой C11 (GCC или Clang).

```sh
cmake -B build
cmake --build build
```

Исполняемый файл: `build/csvreader`.

По умолчанию в Debug-режиме включены санитайзеры AddressSanitizer, LeakSanitizer и UndefinedBehaviorSanitizer. Отключить: `-DSANITIZE=OFF`.

## Запуск

```sh
./csvreader file.csv
```

Результат выводится в stdout в том же CSV-формате. Пустые ячейки обозначаются `EMPTY`. При любой ошибке программа завершается с ненулевым кодом без вывода результата.

## Тесты

Для запуска тестов требуется [CMocka](https://cmocka.org/).

```sh
cmake -B build
cmake --build build
ctest --test-dir build --output-on-failure
```

Тесты включают:
- **unit-тесты** (`tests/test_csv.c`) — 65 тест-кейсов: парсинг заголовка, разбор формул, вычисление, обнаружение циклов, переполнение
- **стресс-тесты** (`tests/test_stress.c`) — случайные таблицы: корректные и некорректные входные данные
- **интеграционные тесты** (`tests/test_binary.sh`) — чёрный ящик: проверка stdout/stderr и кодов возврата

## Коды возврата

| Код | Значение |
|-----|----------|
| `0` | Успех |
| `2` | Ошибка выделения памяти |
| `3` | Ошибка разбора CSV (неверный формат) |
| `4` | Ошибка вычисления (деление на ноль, цикл, переполнение и т.д.) |

## Архитектура

```
include/    — заголовочные файлы
src/        — реализация
  main.c        — точка входа
  Parse.c       — разбор CSV
  Table.c       — хранение таблицы и вычисление формул
  Cell.c        — управление памятью ячеек
  Row.c         — управление строками
  HashStrInt.c  — хеш-таблица: имя столбца → индекс (FNV-1a)
  HashIntRow.c  — хеш-таблица: номер строки → Row*
tests/      — тесты
```

Формулы вычисляются лениво и рекурсивно. Цикличные ссылки обнаруживаются через состояние `CELL_EVALUATING`. Переполнение целых чисел детектируется через `__builtin_*_overflow`.
